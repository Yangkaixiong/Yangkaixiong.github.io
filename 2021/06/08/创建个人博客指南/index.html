1.需求分析

这一部分主要是对为什么要进行个人博客部署进行分析，从功能需求和性能需求两方面来进行。

1.1 功能需求

  • 避免重复性劳动：在学习和工作过程中，我们经常遇到一些这样或那样的问题，此时我们可能会在网上找到相应的解决方法。但是过了一段时间之后，当我们再次碰到类似的问题时，早已忘记以前是怎么解决的了，于是又要到网上去搜，浪费大量的时间和精力。
  • 随时随地可查。
  • 归纳总结：通过记录解决问题的过程，对问题进行复盘。
  • 进一步加深学习的程度：学习的过程就是先学习再实践，再学习，再实践的过程，而向别人讲清楚你所学的知识就是实践方法之一。

1.2 性能需求

  • 方便撰写和发布：既可以在手机上，也可以在电脑上撰写。
  • 安全性好。

2.现状分析

目前，主流的博客主要分静态博客和动态博客两类：所谓动态，就是指有前端有后端，可以登录到后台进行管理。静态博客是纯前端的展示，没有后端。

2.1 动态博客

2.1.1 CSDN、知乎等公共平台
  • 优点：会打字就行、阅读量高。
  • 缺点：限制多。
2.1.2 WordPress等框架
  • 优点：限制少、定制感强。
  • 缺点：笨重、有点门槛。
2.1.3 使用Python等自己开发
  • 优点：功能强大、获得极大成就感。
  • 缺点：门槛高。

2.2 静态博客

2.2.1 使用hexo、hugo、jekyll等框架 + github page
  • 优点：易上手、免费。
  • 缺点：访问慢。
2.2.2 使用hexo、hugo、jekyll等框架 +云服务器
  • 优点：访问速度快。
  • 缺点：要花点money。

2.3 总结

经过以上的分析，本文选用两种方式部署，一种是hexo + 腾讯云服务器，一种是hexo + github page。

3.hexo+腾讯云服务器部署实现

3.1 总体设计

该方法部署主要分为三个部分：本地hexo部分、git部分、云服务器部分。工作原理如图3.1所示。

​ 图3.1

简单来说就是在本地计算机搭建Hexo环境，Hexo通过generate命令将.md文件渲染成静态的html页面，然后Hexo通过deploy命令触发git用户通过公钥免密登陆服务器，进而将静态页面推送到服务器的git仓库（repository）中。然后，服务器再通过钩子（git-hooks） 将静态页面checkout到网站的根目录下，进而实现博客的自动部署。具体过程如图中实线箭头所示。

5.问题汇总

5.1.1 Unable to locate package pcre换源update和upgrade：
  • linux上安装软件分为两种，一种软件源，一种是下载源码然后自己编译。当软件源中没有所需软件时，就会出现上述错误。
5.1.2 Nginx启动时提示，emerg still could not bind():
  • 这是因为端口被占用，可以kill掉nginx进程。
5.1.3 访问nginx返回403：

  • 开了selinux，关闭即可。

  • nginx配置中直接返回了403http状态码。

  • nginx中配置的站点目录内文件没有访问权限。

5.1.4 使用ssh密钥：
  1. 本地生成密钥对。
  2. 设置github上的公钥。
  3. 修改git的remote url为git协议。
5.1.5 hexo使用theme出现“ % extends ‘_layout.swig‘ % % import ‘_macro/post.swig‘ as post_template %“问题：
  • 原因是hexo在5.0之后把swig给删除了需要自己手动安装。
5.1.6 hexo generate时err: template render unknown path：
  • 配置文件有空格。
  • 出现了花括号。
  • 代码没用代码格式。
5.1.7 git@1.117.216.4:/home/git/hexo.git和hexo的区别：
5.1.8 su: Authentication failure

su命令不能切换root，提示su: Authentication failure，只要你sudo passwd root过一次之后，下次再su的时候只要输入密码就可以成功登录了

5.1.9 本地git需要密码：

在云服务器查看相关用户信息

5.1.10 查看是否安装软件：
5.1.11 软件管理包和源码安装时软件安装在哪里：

在哪个目录下使用wget，压缩包就在那里，解压和安装都去解压缩的目录下。

5.1.12 访问时出现403、err_connection_refused：
  • 开放特定端口。
  • 查看error_log文件。
  • 运行软件，开启端口。
5.1.13 git@github Permission denied (publickey)，Could not read from remote repository：

重新设置密钥。

6.文章小结

在采用别人的经验技巧时，一定要明确别人的环境，对于本文的部署博客，尽量保持和我的环境一致，不然容易出问题，如果是小白的话解决起来相当麻烦。

在搭建个人博客前最好先有一个结构型的思维，知道部署博客分为三部分，然后逐一攻破。

在做事的过程中记录好步骤有利于出问题时的回溯检查，尽可能搞懂每一步，不然出错都不知道哪里出错。

再难的问题也敌不过理性地分析。
